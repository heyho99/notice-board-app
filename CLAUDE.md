# プロジェクト指示書（CLAUDE.md）

このファイルは、Bob（Claude Code）が本プロジェクトで作業する際に必ず最初に読むプロジェクト共通ルールです。
すべての作業は、このファイル・`README.md`・`docs/` 配下の仕様書、および `.claude/rules/` 配下のルールに従って行うこと。

---

## 1. プロジェクト概要

- 名称: 社内お知らせ掲示板Webアプリ
- 目的: 社内お知らせの投稿・閲覧・編集・削除ができる、学習用の小規模 Web アプリ
- 認証なし・データ非永続化（インメモリ）の最小構成

詳細な要件は `README.md` を参照。

## 2. 技術スタック（厳守）

以下の構成から外れないこと。新しいライブラリやフレームワークの導入は事前に学習者へ確認すること。

- フロントエンド
  - 生の JavaScript のみ（React / Vue / jQuery などのフレームワーク・ライブラリ禁止）
  - HTML / CSS は素の状態で記述
- バックエンド
  - Python 3.11
  - Flask
  - venv による仮想環境
  - インメモリデータ（SQLite / 外部 DB / ファイル永続化はすべて禁止）

## 3. ドキュメント構成

ドキュメントは `docs/` 配下に集約する。

| ファイル | 役割 | 状態 |
| --- | --- | --- |
| `README.md` | 要件書 | 事前定義済み |
| `docs/data_model.md` | データモデル定義書 | 事前定義済み（変更禁止） |
| `docs/screen_spec.md` | 画面仕様書 | これから作成 |
| `docs/api_spec.md` | API 仕様書 | これから作成 |
| `docs/backend_spec.md` | バックエンド仕様書 | これから作成 |
| `docs/frontend_spec.md` | フロントエンド仕様書 | これから作成 |

`README.md` と `docs/data_model.md` は本プロジェクトの前提であり、勝手に変更しないこと。
変更が必要と判断した場合は、必ず学習者に確認してから行う。

## 4. 開発フロー

仕様 → 実装 の順を守る。仕様書を書かずにいきなり実装に入らない。

1. `README.md` と `docs/data_model.md` を読み、要件と扱うデータを把握する。
2. `docs/screen_spec.md` を作成（画面構成・遷移・各画面の項目・操作）。
3. `docs/api_spec.md` を作成（エンドポイント・リクエスト/レスポンス・エラー）。
4. `docs/backend_spec.md` を作成 → 仕様に沿って Flask アプリを実装。
5. `docs/frontend_spec.md` を作成 → 仕様に沿って HTML / CSS / JS を実装。
6. ローカルで起動し、機能要件を一通り確認する。

各ステップで成果物を作成したら、次のステップに進む前に学習者に内容を確認してもらうこと。

## 5. 環境ルール

- 仮想環境は必ずプロジェクト直下の `venv/` を利用する。
  - 作成: `python -m venv venv`
  - 有効化（Windows / PowerShell）: `.\venv\Scripts\activate`
- `pip install` を行ったら、`requirements.txt` を更新する。
- バックエンドの起動コマンド・フロントエンドの開き方は、実装後に `README.md` 末尾へ追記してよい。

## 6. コーディング方針

- 学習用プロジェクトのため、過剰な抽象化や将来拡張のための仕掛けは入れない。
- コメントは「なぜそうしたか」が非自明な場合のみ最小限。コードの説明的コメントは不要。
- バリデーション・エラーハンドリングは仕様書に書かれた範囲で実装する。仕様外のケースを勝手に増やさない。
- ファイルや関数を新設する前に、既存ファイル内に追加できないか検討する。

## 7. やり取りのルール

- 出力（チャット応答・コードコメント・ドキュメント）はすべて日本語で行う。
- 不明点・要件の解釈に迷う点があれば、勝手に決めず学習者に確認する。
- 大きな変更や、ドキュメント・仕様の前提を覆す変更を行う前には、必ず確認を取る。

## 8. 参照順序

作業時は、以下の順で情報を確認すること。

1. このファイル（`CLAUDE.md`）
2. `.claude/rules/` 配下の各ルール
3. `README.md`
4. `docs/` 配下の関連仕様書
5. 実装ファイル
