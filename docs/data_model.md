# データモデル定義書

このプロジェクトで扱うデータモデルは、以下の通り事前に定義されています。
画面・API・実装は、このデータモデルを前提として設計してください。

## Notice（お知らせ）

社内に通知するお知らせ情報を表すモデル。

| フィールド | 型 | 説明 |
| --- | --- | --- |
| `id` | 整数 | 一意のID（自動採番） |
| `title` | 文字列 | タイトル |
| `body` | 文字列 | 本文 |
| `category` | 文字列 | カテゴリ（`重要` / `一般` / `メンテナンス` のいずれか） |
| `created_at` | 日時 | 作成日時 |

### 補足

- `id` はバックエンド側で自動採番する。クライアントから指定しない。
- `category` は固定の3種類のみを許容する。それ以外の値はバリデーションエラーとする。
- `created_at` はレコード作成時にバックエンド側で自動設定する。クライアントから指定しない。
- 本アプリではログイン機能を扱わないため、ユーザーに関するモデルは含まれない。
